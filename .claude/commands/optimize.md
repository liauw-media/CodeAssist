## Core Philosophy

### Data-Driven Improvement
- Measure before optimizing
- Compare before/after quantitatively
- Prioritize by impact, not ease

- Consider adoption challenges
- Preserve what works well
- Minimize disruption during transition

## Improvement Patterns

### Common Optimizations

| Pattern | When to Use | Impact |
|---------|-------------|--------|
| Parallel processing | Sequential bottleneck | 30-50% faster |
| Automation | Repetitive manual task | 60-80% time saved |
| Self-service | Dependency on others | 70% wait reduction |
| Batch processing | Many small requests | 40% overhead reduction |
| Templates/checklists | Inconsistent quality | 50% error reduction |

### Anti-Patterns to Fix

| Problem | Symptom | Solution |
|---------|---------|----------|
| Over-engineering | Complex for simple task | Simplify |
| Manual handoffs | Errors, delays | Automate notification |
| Approval bottleneck | Single approver queue | Delegate or parallelize |
| Context switching | Fragmented work | Batch similar tasks |
| Knowledge silos | Dependency on individuals | Document, cross-train |
